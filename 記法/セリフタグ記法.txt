  セリフファイル（.txt）の書き方とタグ一覧
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 基本ルール

  - 1行 = 1セリフ
  - "#" で始まる行はコメント（無視される）
  - 空行は無視される
  - セリフは表示時にランダムで1行が選ばれる

  例:
    # 朝のセリフ
    おはよう、{userName}！今日もよろしくね！
    今日は{weatherEmoji}だね。
    {consecutiveDays}日連続！すごい！[v:favorability+2]

■ 変数展開  {変数名}

  セリフ中に波括弧で変数名を書くと、実行時に値に置換される。
  標準変数・カスタム変数どちらも使える。

    こんにちは{userName}！好感度は{favorability}だよ。
    今日は{weatherEmoji}{weatherCode}。気温{temperature}℃だね。

  → 使える変数の一覧は「標準変数一覧.txt」を参照

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ 三項演算子  {条件 ? 真値 : 偽値}

  条件式を評価し、結果に応じて表示テキストを切り替える。
  条件部には標準変数・カスタム変数・文字列リテラルが使える
  （emotions の "if" と同じ SafeExpressionEvaluator）。

    今日は{weatherEmoji}だね。{temperatureFeeling == "寒い" ? 寒いね : いい天気だね}！
    {isLowBattery ? 充電した方がいいよ : バッテリー余裕あり}

  注意:
    - 条件部に「?」を含めないこと
    - 真値・偽値の中に「:」や「}」を含めないこと（単純な文字列向け）
    - 評価に失敗した場合は偽値側が表示される

■ 感情タグ  [emotion:感情名]  /  [e:感情名]

  そのセリフを表示するときに使用する画像を指定する。
  images に定義した感情名を使う。
  [e:感情名] は短縮形で、[emotion:感情名] と同じ意味。

    うれしいな！[emotion:happy]
    たのしい！[e:happy]
    ちょっと怒ってるよ。[emotion:angry]

  - セリフ1行につき1つまで有効（複数書いた場合は最初の1つ）
  - 表示テキストからは除去される
  - emotions ルールより、セリフ内タグが優先される

  【感情名の制限】
    感情名は英数字とアンダースコアのみ、1〜30文字:
      使える例: happy / sad / sleepy / very_happy
      使えない例: うれしい / happy-face （タグとして認識されない）

    images のキーもこの形式に揃えること。

■ 変数操作タグ  [v:変数名 操作]  /  [var: 変数名 操作]

  どちらの記法でも同じように動く。
  セリフが表示されると同時にカスタム変数を操作する。
  1行に複数書いてもよい。
  空白の有無はどちらでも可（例: [v:favorability+2] と [v: favorability + 2]）。

  ──────────────────────────────
  【NUMBER型の操作】

    [v:favorability+2]      favorability += 2
    [v:favorability-1]      favorability -= 1
    [v:favorability*2]      favorability *= 2
    [v:favorability/2]      favorability /= 2  （※0での除算は無視されます）
    [v:favorability=50]     favorability = 50

  ──────────────────────────────
  【STRING型の操作】

    [v:mood=happy]          mood = "happy"
    [v:mood=うれしい]        mood = "うれしい"  ← 日本語も可（右辺のみ）

    - 等号の右辺（RHS）は [ ] を含まない最大50文字の任意文字列を設定可能
    - マルチバイト文字（日本語など）も正常にパースされる
    - options リストに含まれていない値を指定するとスキップ（警告ログ出力）
    - options 自体が未定義の STRING 型には代入できない（後述）

  ──────────────────────────────
  【BOOLEAN型の操作】

    [v:isAngry=true]        isAngry = true
    [v:isAngry=false]       isAngry = false
    [v:isAngry=toggle]      isAngry = !isAngry  （現在値を反転）

  ──────────────────────────────
  【複数同時操作】

    ありがとう！[v:favorability+5][v:isAngry=false][emotion:happy]

  ──────────────────────────────
  【型の間違いはスキップ】

    NUMBER型に [v:xxx=happy]    → スキップ
    STRING型に [v:xxx=toggle]   → スキップ
    BOOLEAN型に [v:xxx=toggle]  → 正常動作

  ──────────────────────────────
  【変数名の制限】

    操作対象の変数名は英数字とアンダースコアのみ
    （customVariables のキーもこの形式にすること）

■ 改行タグ  {br} / \n

  セリフ内で改行したい場合。どちらを使っても同じ結果になる。
  {br} は大文字小文字を区別しない。

    1行目のテキスト{br}2行目のテキスト
    1行目のテキスト\n2行目のテキスト

■ タグ一覧まとめ

  タグ                              概要
  ─────────────────────────────────────────────
  {変数名}                          変数値に置換（標準・カスタム共通）
  {条件 ? 真値 : 偽値}              三項演算子（条件で表示を分岐）
  [emotion:感情名] / [e:感情名]     その行の表示画像を指定（英数字_のみ）
  [v:変数名+数値]                   NUMBER加算  ※[var: ...] も同じ
  [v:変数名-数値]                   NUMBER減算
  [v:変数名*数値]                   NUMBER乗算
  [v:変数名/数値]                   NUMBER除算
  [v:変数名=数値]                   NUMBER代入
  [v:変数名=文字列]                 STRING代入（最大50文字、[ ] 不可、マルチバイトOK）
  [v:変数名=true/false]             BOOLEAN代入
  [v:変数名=toggle]                 BOOLEAN反転
  {br} / \n                         改行
